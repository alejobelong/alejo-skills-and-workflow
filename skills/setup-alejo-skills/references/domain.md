# Domain Docs

Purpose: give Alejo skills one place to find project language before writing PRDs, SADs, issues, tests, or implementation names.

Use:

- root `CONTEXT.md` for one-context repos
- `CONTEXT-MAP.md` for multi-context repos
- `docs/adr/` for durable decisions

`alejo-questions` creates or updates these docs when it resolves new terms or decisions.
